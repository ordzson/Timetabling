# ADR 0005 - Entrega completa sin MVP por fases

## Estado

Aceptado

## Contexto

El entregable del curso debe ser sistema completo. No se entregara solo motor o prototipo aislado.

## Decision

Planificar una entrega final completa con backend, motor, importacion, UI, reportes, roles, edicion manual y despliegue.

## Consecuencias

- El alcance se organiza por tareas pequenas, no por producto parcial.
- Cada parte debe tener criterio de salida verificable.
- Se evitan funcionalidades especulativas fuera del problema.
- La prioridad de implementacion sigue dependencias tecnicas, no demos aisladas.
